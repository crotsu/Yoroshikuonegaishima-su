# 提出状況確認スクリプト実装計画

## 目的
`QUESTION_ROOT` 内の `*.md` ファイルをパースして課題一覧を取得し、実行ユーザーの提出状況を `SUBMISSION_BASE/<ユーザー名>/<dir>/<file>` の存在確認で判定、提出済み数・未提出数・バトルポイント合計を表示するスクリプトを実装する。

## アーキテクチャ
- エントリポイントは `scouter.py` とする。
- 設定ファイル読み込み、課題定義ファイルパース、提出状況確認、集計・表示を関数分割する。
- 実処理は `pathlib` を用いたローカルファイル存在確認のみで完結させる。
- パスは `config.py` に外部化し、課題リスト・バトルポイントは `QUESTION_ROOT/*.md` から読み込む。

## 技術スタック
- Python 3
- 標準ライブラリのみ

## ファイル構成
- `scouter.py` — スクリプト本体
- `config.py` — パス設定（gitignore 対象）
- `config.py.example` — 設定テンプレート（`QUESTION_ROOT` のサンプルを追記）
- `docs/specs/check-assignment.md`
- `docs/plans/check-assignment.md`
- `tests/test_scouter.py`

## 課題定義ファイルの形式
`QUESTION_ROOT/j2pro0408.md` の例:
```
No0408_1.c, 123
No0408_2.c, 1251
```
- 各行: `<ファイル名>, <バトルポイント>`
- 空行・カンマなし行は無視する。

## 提出スクリプトとの兼ね合い
- `yorosikuonegaishima-su.py` も同じ `*.md` ファイルを読む。
- 現在は1行1ファイル名の形式だが、新形式（`filename, point`）に変更する。
- 提出スクリプト側はカンマ区切りの先頭要素だけを取得するよう修正が必要。

## 実装手順
1. `config.py` を読み込み、`QUESTION_ROOT` と `SUBMISSION_BASE` を取得する。
   - `config.py` が存在しない場合はエラーメッセージを表示して終了する。
2. 環境変数 `USER` からユーザーIDを取得し、先頭に表示する。
3. `QUESTION_ROOT/*.md` をファイル名順に列挙する。
4. 各 `.md` ファイルをパースし、`(ファイル名, バトルポイント)` のリストを生成する。
5. 課題ごとに `SUBMISSION_BASE / <ユーザーID> / <dir> / <file>` の存在を確認する。
6. `O.K.` または空白で各行を表示する。
7. 提出済み件数・総件数・バトルポイント合計を集計して表示する。

## 検証項目
- 全課題未提出の状態で `0/<総数>` と `Battle Point=0` が表示される。
- 一部提出済みの場合に `O.K.` と集計が正しく表示される。
- 全課題提出済みの場合に全行 `O.K.` となり合計ポイントが正しい。
- `config.py` が存在しない場合にエラーメッセージを表示して終了する。
- `QUESTION_ROOT` に `.md` ファイルが存在しない場合に `0/0` と `Battle Point=0` が表示される。
- 複数の `.md` ファイルがファイル名順に処理される。
