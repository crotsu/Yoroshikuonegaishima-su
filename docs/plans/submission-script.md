# 提出スクリプト実装計画

## 目的
課題ディレクトリ内の `*.c` を設定ファイルと照合し、一致するものだけを `SUBMISSION_BASE/<ユーザー名>/<課題ディレクトリ名>` へコピーする提出スクリプトを実装する。

## アーキテクチャ
- エントリポイントは `yorosikuonegaishima-su.py` とする。
- CLI 引数検証、設定ファイル読み込み、提出処理を関数分割する。
- 実処理は `pathlib` と `shutil` を用いてローカルファイル操作だけで完結させる。
- パスは `config.py` に外部化し、リポジトリには含めない。

## 技術スタック
- Python 3
- 標準ライブラリのみ

## ファイル構成
- `yorosikuonegaishima-su.py` — スクリプト本体
- `config.py` — パス設定（gitignore 対象）
- `config.py.example` — 設定テンプレート
- `docs/specs/submission-script.md`
- `docs/plans/submission-script.md`

## config.py の配置
- 本番環境: `/home/class/j2/prog/.send/j25/questions/config.py`
- スクリプトをコピーした後、上記パスに `config.py` を作成する。

## 実装手順
1. CLI の引数数を検証し、使い方メッセージを実装する。
2. `config.py` を読み込み、`QUESTION_ROOT` と `SUBMISSION_BASE` を取得する。
3. 指定ディレクトリの存在確認を実装する。
4. `QUESTION_ROOT/<課題ディレクトリ名>.md` の設定ファイル存在確認を実装する。
5. 実行ユーザー名を取得し、`SUBMISSION_BASE/<ユーザー名>/<課題ディレクトリ名>` の提出先パスを組み立てる。
6. 提出先ディレクトリの存在確認を実装する。
7. 設定ファイルを読み込み、受理可能なファイル名一覧を集合として保持する。
8. 指定ディレクトリ直下の `*.c` を列挙し、ファイル名一致判定を実装する。
9. 一致したファイルを提出先ディレクトリへコピーし、新規提出と上書き提出のメッセージを分ける。
10. 不一致ファイル、受理ファイルなし、提出先ディレクトリ未存在のメッセージを実装する。

## 検証項目
- 引数なしで使い方メッセージが出る。
- 存在しない課題ディレクトリを指定するとエラー終了する。
- 設定ファイルがない場合にエラーメッセージを出して終了する。
- 提出先ディレクトリが存在しない場合にエラーメッセージを出して終了する。
- 一致する `*.c` だけが提出先にコピーされる。
- 初回コピー時に `新規に提出しました。` が表示される。
- 同名ファイル再提出時に `上書きしました。` が表示される。
- 不一致の `*.c` に対して拒否メッセージが表示される。
- `*.c` が 1 件もない場合にメッセージが表示される。
