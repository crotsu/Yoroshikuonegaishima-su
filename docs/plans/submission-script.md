# 提出スクリプト実装計画

## 目的
課題ディレクトリ内の `*.c` を設定ファイルと照合し、一致するものだけを `submissions` へコピーする提出スクリプトを実装する。

## アーキテクチャ
- エントリポイントは `yorosikuonegaishima-su.py` とする。
- CLI 引数検証、設定ファイル読み込み、提出処理を関数分割する。
- 実処理は `pathlib` と `shutil` を用いてローカルファイル操作だけで完結させる。

## 技術スタック
- Python 3
- 標準ライブラリのみ

## ファイル構成
- `yorosikuonegaishima-su.py`
- `docs/specs/submission-script.md`
- `docs/plans/submission-script.md`

## 実装手順
1. CLI の引数数を検証し、使い方メッセージを実装する。
2. 指定ディレクトリの存在確認を実装する。
3. `/home/class/question/<課題ディレクトリ名>` の設定ファイル存在確認を実装する。
4. 設定ファイルを読み込み、受理可能なファイル名一覧を集合として保持する。
5. 指定ディレクトリ直下の `*.c` を列挙し、ファイル名一致判定を実装する。
6. 一致したファイルを `./submissions` にコピーし、新規提出と上書き提出のメッセージを分ける。
7. 不一致ファイルと受理ファイルなしのメッセージを実装する。
8. 正常系と異常系をローカルで検証する。

## 検証項目
- 引数なしで使い方メッセージが出る。
- 存在しない課題ディレクトリを指定するとエラー終了する。
- 設定ファイルがない場合にエラーメッセージを出して終了する。
- 一致する `*.c` だけが `submissions` にコピーされる。
- 初回コピー時に `新規に提出しました。` が表示される。
- 同名ファイル再提出時に `上書きしました。` が表示される。
- 不一致の `*.c` に対して拒否メッセージが表示される。
- `*.c` が 1 件もない場合にメッセージが表示される。
